---
title: Credentials API
description: Email, phone, and personhood credential verification endpoints.
---

## Start email verification

<ApiEndpoint method="POST" path="/v1/credentials/email/start" auth="atap:manage" description="Send a verification code to an email address." />

## Complete email verification

<ApiEndpoint method="POST" path="/v1/credentials/email/verify" auth="atap:manage" description="Submit the verification code to issue an ATAPEmailVerification credential." />

## Start phone verification

<ApiEndpoint method="POST" path="/v1/credentials/phone/start" auth="atap:manage" description="Send a verification code to a phone number." />

## Submit personhood proof

<ApiEndpoint method="POST" path="/v1/credentials/personhood" auth="atap:manage" description="Submit proof-of-personhood to issue an ATAPPersonhood credential." />

## List credentials

<ApiEndpoint method="GET" path="/v1/credentials" auth="atap:manage" description="List all credentials for the authenticated entity." />

## Credential status list

<ApiEndpoint method="GET" path="/v1/credentials/status/{list-id}" auth="None" description="Public Bitstring Status List for credential revocation checks." />

Verifiers query this endpoint to check whether a credential has been revoked.
