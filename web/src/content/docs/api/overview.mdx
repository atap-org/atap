The ATAP API is a REST API for entity management, credential issuance, revocation, and DIDComm messaging. Approvals themselves are transported via DIDComm — not via REST.

## Base URL

Declared in `/.well-known/atap.json`. The reference implementation uses:

- **Production:** `https://api.atap.dev/v1`
- **Sandbox:** `https://api.sandbox.atap.dev/v1`

## Authentication

Authenticated endpoints require **OAuth 2.1 + DPoP** (RFC 9449). Each request includes:
1. A `DPoP` header with a proof JWT
2. An `Authorization: DPoP <token>` header

See [OAuth & DPoP](/docs/api/oauth) for token acquisition.

## Scopes

| Scope | Permits |
|-------|---------|
| `atap:inbox` | Read inbox, SSE stream |
| `atap:send` | Send DIDComm messages |
| `atap:revoke` | Submit approval revocations |
| `atap:manage` | Manage credentials, key rotation |

## Error format

Errors follow RFC 7807 (Problem Details):

```json
{
  "type": "https://atap.dev/errors/trust-level-insufficient",
  "title": "Insufficient trust level",
  "status": 403,
  "detail": "This endpoint requires trust level 2.",
  "instance": "/v1/credentials/personhood"
}
```
