**Test helper — create a minimal test handler with rate limit middleware:**

```go
// newRateLimitTestHandler creates a minimal Handler + Fiber app with only the rate limit middleware
// and a dummy 200 OK handler. Uses a real Redis connection (skips if unavailable).
func newRateLimitTestHandler(t *testing.T) (*Handler, *fiber.App, *rateLimitConfig) {
    t.Helper()

    // Connect to Redis (same as approvals_test.go pattern)
    rdb := redis.NewClient(&redis.Options{Addr: "localhost:6379"})
    ctx := context.Background()
    if err := rdb.Ping(ctx).Err(); err != nil {
        t.Skipf("Redis not available (%v), skipping rate limit test", err)
    }
    t.Cleanup(func() { rdb.Close() })

    log := zerolog.Nop()

    cfg := &rateLimitConfig{
        publicRPM: 30,
        authRPM:   120,
    }

    h := &Handler{
        redis: rdb,
        log:   log,
    }

    app := fiber.New()
    app.Use(h.RateLimitMiddleware(cfg))

    // Dummy handler — returns 200 OK for any route
    app.Get("/v1/health", func(c *fiber.Ctx) error { return c.SendStatus(200) })
    app.Get("/.well-known/atap.json", func(c *fiber.Ctx) error { return c.SendStatus(200) })
    app.All("/*", func(c *fiber.Ctx) error { return c.SendStatus(200) })

    return h, app, cfg
}
```

**IMPORTANT:** Route registration order matters for Fiber. The exempt paths (/v1/health, /.well-known/atap.json) are registered BEFORE the catch-all, but the middleware runs on ALL routes. The middleware itself checks the path and short-circuits for exempt paths. The dummy handler routes just ensure Fiber has a handler to call.
