    **Create `platform/internal/api/did.go`:**
    - `ResolveDID(c *fiber.Ctx) error` -- GET /:type/:id/did.json handler:
      1. Extract `type` and `id` from URL params
      2. Validate type is one of: agent, machine, human, org. Return 404 for invalid types.
      3. Look up entity by ID in store. Return 404 if not found.
      4. Verify entity.Type matches the URL type parameter. Return 404 if mismatch (prevents cross-type resolution).
      5. Fetch all key versions for the entity from store.
      6. Call crypto.BuildDIDDocument(entity, keyVersions, config.PlatformDomain)
      7. Set `Content-Type: application/did+ld+json` (per Pitfall 7 in RESEARCH.md -- DO NOT use Fiber's c.JSON which defaults to application/json)
      8. Marshal DID Document to JSON and write response

    **DID Document structure (from RESEARCH.md Pattern 1):**
    ```json
    {
      "@context": [
        "https://www.w3.org/ns/did/v1",
        "https://w3id.org/security/suites/ed25519-2020/v1",
        "https://atap.dev/ns/v1"
      ],
      "id": "did:web:atap.app:agent:01abc",
      "verificationMethod": [{
        "id": "did:web:atap.app:agent:01abc#key-1",
        "type": "Ed25519VerificationKey2020",
        "controller": "did:web:atap.app:agent:01abc",
        "publicKeyMultibase": "z6Mkf..."
      }],
      "authentication": ["did:web:atap.app:agent:01abc#key-1"],
      "assertionMethod": ["did:web:atap.app:agent:01abc#key-1"],
      "atap:type": "agent",
      "atap:principal": "did:web:atap.app:human:abcdef1234567890"
    }
    ```

    For rotated keys (DID-07): include all key versions in verificationMethod array. Only the active key (valid_until IS NULL) appears in authentication and assertionMethod. Expired keys have their validity period in the verification method entry.
