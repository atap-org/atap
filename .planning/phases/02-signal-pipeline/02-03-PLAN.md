**3. Create webhook.go** with WebhookWorker:

```go
type WebhookWorker struct {
    queue       chan WebhookJob
    httpClient  *http.Client
    store       WebhookStore
    platformKey ed25519.PrivateKey
    log         zerolog.Logger
}

type WebhookJob struct {
    SignalID   string
    EntityID   string
    WebhookURL string
    Payload    []byte
    Attempt    int
}

var retryDelays = []time.Duration{
    1 * time.Second, 5 * time.Second, 30 * time.Second,
    5 * time.Minute, 30 * time.Minute,
}
```

- `NewWebhookWorker(store WebhookStore, platformKey ed25519.PrivateKey, log zerolog.Logger) *WebhookWorker` — creates worker with buffered channel (1000 capacity), http.Client with 10s timeout
- `Start(ctx context.Context, workers int)` — launches N worker goroutines
- `Enqueue(job WebhookJob)` — non-blocking send to channel (drop if full, log warning)
- `worker(ctx)` — reads from queue, calls deliver()
- `deliver(ctx, job)`:
  a. Sign payload with platform Ed25519 key: `sig := crypto.Sign(w.platformKey, job.Payload)`, base64 encode
  b. Create HTTP POST request to job.WebhookURL with job.Payload as body, Content-Type application/json
  c. Set `X-ATAP-Signature` header with base64-encoded signature
  d. Set `X-ATAP-Signal-ID` header with job.SignalID
  e. Execute request, record attempt via store.SaveDeliveryAttempt
  f. If success (2xx): update signal delivery_status to "delivered"
  g. If failure and attempt < 5: compute next retry delay from retryDelays[attempt], save attempt with next_retry_at
  h. If failure and attempt >= 4 (0-indexed, so 5th attempt): update signal delivery_status to "failed"

- `StartRetryPoller(ctx context.Context, interval time.Duration)` — background goroutine that polls GetPendingRetries every interval (e.g., 5 seconds), re-enqueues due jobs
- `StartCleanupJob(ctx context.Context, interval time.Duration)` — background goroutine that calls CleanupDeliveryAttempts(olderThan: 24h ago) every hour

**5. Implement API handlers in api.go:**

`SetWebhook` (POST /v1/entities/:entityId/webhook):
- Auth required. Verify authenticated entity matches :entityId
- Parse SetWebhookRequest { URL string }
- Validate URL is non-empty and starts with https:// (or http:// for dev)
- Call webhookStore.SetWebhookConfig
- Return 200 with { webhook_url, updated_at }

`CreateChannel` (POST /v1/entities/:entityId/channels):
- Auth required. Verify authenticated entity matches :entityId
- Parse CreateChannelRequest { Label, Tags, Type, TrusteeID }
- Validate Type is "trusted" or "open"
- If trusted: validate TrusteeID is provided and entity exists (entityStore.GetEntity)
- If open: generate random Basic Auth password (32 bytes, base64url encoded), bcrypt hash it, store hash in channel
- Generate channel ID with crypto.NewChannelID()
- Construct webhook URL: `{platform_domain}/v1/channels/{channel-id}/signals`
- Call channelStore.CreateChannel
- Return 201. For open channels, include BasicAuthPassword in response (returned once, never again). For trusted channels, omit credentials.

`ListChannels` (GET /v1/entities/:entityId/channels):
- Auth required. Verify authenticated entity matches :entityId
- Call channelStore.ListChannels
- Return 200 with array of channels

`RevokeChannel` (DELETE /v1/entities/:entityId/channels/:channelId):
- Auth required. Verify authenticated entity matches :entityId
- Verify channel belongs to entity
- Call channelStore.RevokeChannel
- Return 204

`ChannelInbound` (POST /v1/channels/:channelId/signals):
- NO auth middleware (this is a public webhook endpoint)
- But for trusted channels: verify Ed25519 signature via existing auth middleware logic (check Authorization header, verify signer is the trustee)
- For open channels: verify Basic Auth credentials (Authorization: Basic ..., decode, bcrypt compare against stored hash)
- Validate channel is active
- Check payload size <= 64KB
- Wrap raw payload into an ATAP Signal:
  - ID: crypto.NewSignalID()
  - Route: origin = channel trustee URI (trusted) or "external://{channel-id}" (open), target = channel owner entity URI, channel = channel ID
  - Trust: level 0 for open channels, signer trust level for trusted
  - Signal body: type = "channel.inbound", data = raw payload
  - Context: source = "agent" (trusted) or "external" (open), trust_level 0 for external
- Save signal via signalStore.SaveSignal
- Publish to Redis inbox
- Increment channel signal count
- Return 202
