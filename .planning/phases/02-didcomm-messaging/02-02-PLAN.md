<objective>
Extend the entity and DID infrastructure for DIDComm messaging: X25519 key generation, DID Document keyAgreement + service endpoints, message queue schema, and server-level DIDComm identity.

Purpose: DIDComm requires X25519 key agreement keys (separate from Ed25519 signing keys) and a DIDCommMessaging service endpoint in each entity's DID Document. The server itself needs its own DID and X25519 keypair to act as a trusted `via` participant (MSG-03). The message queue schema enables offline delivery. Without these, no entity can receive encrypted messages.

Output: Migration 010 with X25519 columns and didcomm_messages table, updated entity registration and DID Document resolution, server DID identity with X25519 keypair.
</objective>

```sql
CREATE TABLE didcomm_messages (
    id             TEXT PRIMARY KEY,           -- "msg_" + ULID
    recipient_did  TEXT NOT NULL,
    sender_did     TEXT,                        -- NULL for anoncrypt
    message_type   TEXT,                        -- type URI (only populated if server is recipient)
    payload        BYTEA NOT NULL,              -- raw JWE bytes (encrypted, opaque to server)
    state          TEXT NOT NULL DEFAULT 'pending',  -- pending | delivered | expired
    created_at     TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at     TIMESTAMPTZ,
    delivered_at   TIMESTAMPTZ
);

CREATE INDEX idx_didcomm_messages_recipient ON didcomm_messages(recipient_did, state);
CREATE INDEX idx_didcomm_messages_expires ON didcomm_messages(expires_at) WHERE state = 'pending';
```
