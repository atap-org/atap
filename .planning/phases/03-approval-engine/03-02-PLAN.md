<objective>
Rework template model to Adaptive Cards format and extend DIDComm handler for revocation message processing.

Purpose: The spec v1.0-rc1 changed templates from custom JSON to Microsoft Adaptive Cards format (opaque `card` field + `proof`). The DIDComm handler must process `approval/1.0/revoke` messages by storing a revocation entry and forwarding to the `via` system. This completes the revocation flow: approver sends revoke via DIDComm -> server stores in revocation list + forwards to via.

Output: Updated template library code, DIDComm revocation handler, full test coverage.
</objective>
