**In approvals.go (new file):**

Create all 6 handlers on `*Handler` receiver. Each follows the established pattern: parse request, validate, call store, return JSON response. Use `problem()` for errors.

1. **CreateApproval** (`POST /v1/approvals`):
   - Parse JSON body: expects `to` (DID), `via` (optional DID), `subject` (object), `template_url` (optional), `parent` (optional approval ID), `valid_until` (optional ISO 8601).
   - `from` is the authenticated entity's DID (from DPoP auth context -- extract via `c.Locals("entity_did")` or however the auth middleware stores it; check the existing pattern in entities.go for how authenticated entity is accessed).
   - Generate approval ID via `crypto.NewApprovalID()`.
   - Set `atap_approval: "1"`, `created_at: time.Now().UTC()`.
   - If `valid_until` is set, clamp via `approval.ClampValidUntil(validUntil, h.config.MaxApprovalTTL)`.
   - Validate `to` entity exists via `h.entityStore.GetEntityByDID(ctx, to)`. Return 404 if not found.
   - If `via` is set (three-party): validate `via` entity exists. For Phase 3, the server IS the `via` (server DID = `did:web:{domain}:server:platform`). Verify the `via` DID matches the server's DID.
   - If `parent` is set: validate parent approval exists and is not in a terminal state.
   - Initialize `Signatures: map[string]string{}`.
   - Sign the approval as `from`: look up the authenticated entity's private key. Call `approval.SignApproval(apr, fromPrivKey, fromKeyID)`. Set `Signatures["from"] = jws`.
   - If three-party (`via` present): Server co-signs as `via`. Call `approval.SignApproval(apr, h.platformKey, serverKeyID)`. Set `Signatures["via"] = jws`. If `template_url` is set, optionally fetch and verify the template (but do not block creation on template fetch failure -- template verification is a client-side concern per spec 11.1).
   - Set state to "requested".
   - Store via `h.approvalStore.CreateApproval(ctx, apr)`.
   - Dispatch DIDComm message: For two-party, send `TypeApprovalRequest` from `from` to `to`. For three-party, send `TypeApprovalRequest` from `via` to `to` (since `via` has already co-signed). Use the DIDComm mediator to queue the message.
   - Return 201 with the approval document JSON.

   IMPORTANT for DIDComm dispatch: The existing mediator in `didcomm/mediator.go` handles message routing. Create a `PlaintextMessage` using `didcomm.NewMessage()` with the approval as an attachment (`data.json`), then queue it via `h.messageStore.QueueMessage()`. The body should contain `approval_id` and status for lightweight reference per research recommendation.

   IMPORTANT for `from` private key access: The authenticated entity's private key is NOT available in the handler (private keys stay on device per CLAUDE.md crypto rules). The `from` entity MUST sign the approval client-side before submitting. Revise: The POST body should include a pre-signed `from_signature` (JWS compact detached). The server verifies it using `approval.VerifyApprovalSignature` against the entity's public key from the DB. The server then adds the `via` signature (if three-party) using `h.platformKey`.

2. **RespondApproval** (`POST /v1/approvals/{id}/respond`):
   - Parse body: `status` ("approved" | "declined"), `signature` (JWS compact detached from `to`).
   - Get approval from store. Return 404 if not found.
   - Verify authenticated entity's DID matches approval.To. Return 403 if mismatch.
   - Validate state transition: `approval.ValidateTransition(apr.State, status)`. Return 409 if invalid.
   - Verify the `to` signature: look up `to` entity's public key, call `approval.VerifyApprovalSignature(apr, signature, toPubKey)`. Return 400 if invalid.
   - Set `Signatures["to"] = signature`.
   - If `status == "approved"` and `ValidUntil == nil` (one-time): transition to "approved" now, will transition to "consumed" on first use (APR-09).
   - Update state via store.
   - Dispatch DIDComm `TypeApprovalResponse` to `from` (and `via` if present).
   - Return 200 with updated approval.

3. **GetApproval** (`GET /v1/approvals/{id}`):
   - Get by ID from store. Return 404 if not found.
   - Verify authenticated entity is a party (from, to, or via). Return 403 if not.
   - Return 200 with full approval document JSON.

4. **GetApprovalStatus** (`GET /v1/approvals/{id}/status`):
   - Get by ID. Return 404 if not found.
   - Check: state is "approved", valid_until has not passed, parent (if any) is valid.
   - Return 200 with `{ "approval_id": "...", "valid": true/false, "checked_at": "..." }`.

5. **ListApprovals** (`GET /v1/approvals`):
   - Authenticated entity's DID from auth context.
   - Parse `limit` (default 20, max 100) and `offset` (default 0) query params.
   - Call store.ListApprovals.
   - Return 200 with JSON array.

6. **RevokeApproval** (`DELETE /v1/approvals/{id}`):
   - Get approval. Return 404 if not found.
   - Verify authenticated entity's DID matches approval.To (only approver can revoke). Return 403 if not.
   - Verify approval is in "approved" state and has valid_until set (persistent). Return 409 if one-time or not approved.
   - Call `store.RevokeWithChildren(ctx, id)` to cascade. Return 204.
   - Dispatch DIDComm `TypeApprovalRevoke` to `from` (and `via` if present).
