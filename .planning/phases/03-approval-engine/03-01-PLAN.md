**Migration 011_approvals.up.sql:**
Create the approvals table exactly as specified in 03-RESEARCH.md "Database Schema" section:
```sql
CREATE TABLE approvals (
    id           TEXT PRIMARY KEY,
    state        TEXT NOT NULL DEFAULT 'requested',
    from_did     TEXT NOT NULL,
    to_did       TEXT NOT NULL,
    via_did      TEXT,
    parent_id    TEXT REFERENCES approvals(id),
    document     JSONB NOT NULL,
    created_at   TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    valid_until  TIMESTAMPTZ,
    responded_at TIMESTAMPTZ,
    updated_at   TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_approvals_from ON approvals(from_did, state);
CREATE INDEX idx_approvals_to ON approvals(to_did, state);
CREATE INDEX idx_approvals_via ON approvals(via_did) WHERE via_did IS NOT NULL;
CREATE INDEX idx_approvals_parent ON approvals(parent_id) WHERE parent_id IS NOT NULL;
CREATE INDEX idx_approvals_expires ON approvals(valid_until)
    WHERE state = 'approved' AND valid_until IS NOT NULL;
```

**store/approvals.go:**
Implement on `*Store` receiver (consistent with store.go pattern):

1. `CreateApproval(ctx context.Context, a *models.Approval) error` -- INSERT into approvals. Serialize the full Approval struct (excluding server-side fields) as JSONB into `document` column. Use `json.Marshal` for the document column. Set `state` from `a.State` (default "requested" if empty).

2. `GetApproval(ctx context.Context, id string) (*models.Approval, error)` -- SELECT by id. Read `document` JSONB back into `*models.Approval` via `json.Unmarshal`. Overlay server-side fields (state, responded_at, updated_at) from their columns. Return nil, nil on pgx.ErrNoRows (matching existing pattern).

3. `UpdateApprovalState(ctx context.Context, id, state string, respondedAt *time.Time) error` -- UPDATE state, responded_at, updated_at WHERE id = $1. Return `fmt.Errorf("update approval state: %w", err)` on failure.

4. `ConsumeApproval(ctx context.Context, id string) (consumed bool, err error)` -- Atomically transition a one-time approval from "approved" to "consumed" (APR-09). Use UPDATE ... WHERE id = $1 AND state = 'approved' AND valid_until IS NULL, SET state = 'consumed', updated_at = NOW(). Check rows affected: if 1, return (true, nil) -- approval was consumed. If 0, return (false, nil) -- approval was already consumed or not one-time. This atomic check-and-set prevents race conditions on concurrent consume attempts.

5. `ListApprovals(ctx context.Context, entityDID string, limit, offset int) ([]models.Approval, error)` -- SELECT WHERE from_did = $1 OR to_did = $1 OR via_did = $1, ORDER BY created_at DESC, LIMIT/OFFSET. For each row, unmarshal document JSONB and overlay server-side fields.

6. `GetChildApprovals(ctx context.Context, parentID string) ([]models.Approval, error)` -- Use recursive CTE to find ALL descendants (not just direct children) per APR-11. Mark all as revoked in a single UPDATE (for use by revocation logic):
   ```sql
   WITH RECURSIVE descendants AS (
       SELECT id FROM approvals WHERE parent_id = $1
       UNION ALL
       SELECT a.id FROM approvals a JOIN descendants d ON a.parent_id = d.id
   )
   SELECT id, state, document, ... FROM approvals WHERE id IN (SELECT id FROM descendants)
   ```

7. `RevokeWithChildren(ctx context.Context, parentID string) error` -- Use same recursive CTE but UPDATE state = 'revoked', updated_at = NOW() for the parent AND all descendants in a single statement.

8. `CleanupExpiredApprovals(ctx context.Context) (int64, error)` -- UPDATE state = 'expired', updated_at = NOW() WHERE state = 'requested' AND created_at < NOW() - INTERVAL '24 hours'. Return rows affected. (Following the CleanupExpiredMessages pattern from store/messages.go.)
