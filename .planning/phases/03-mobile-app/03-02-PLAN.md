<interfaces>
<!-- Go auth middleware signature format (Dart must match this exactly) -->

From platform/internal/api/api.go AuthMiddleware:
The Go auth middleware expects Authorization header format:
```
Signature keyId="{key_id}",algorithm="ed25519",headers="(request-target) x-atap-timestamp",signature="{base64url_no_padding}"
```

Signable payload is: `{METHOD} {path} {timestamp}` where timestamp is RFC3339 UTC.
X-ATAP-Timestamp header must be present.

From platform/internal/crypto/crypto.go:
```go
func DeriveHumanID(publicKey ed25519.PublicKey) string {
    hash := sha256.Sum256(publicKey)
    encoded := base32.StdEncoding.EncodeToString(hash[:])
    return strings.ToLower(encoded[:16])
}
```

CRITICAL: Dart implementation of DeriveHumanID must produce identical output to Go.
- Go uses standard base32 (RFC 4648, alphabet A-Z2-7, with padding)
- Take first 16 chars BEFORE lowercasing? No: Go lowercases the result of encoded[:16]
- Actually: base32 output is uppercase, then [:16], then lowercase

Key format notes:
- Go uses `base64.RawURLEncoding` (no padding) for signatures
- Dart `base64Url.encode()` adds padding -- MUST strip trailing '='
