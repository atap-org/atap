Create `platform/internal/didcomm/envelope.go`:

**CRITICAL algorithm details from research (do NOT deviate):**
- `alg`: `ECDH-1PU+A256KW` (key wrapping)
- `enc`: `A256CBC-HS512` (content encryption — NOT XC20P, see research Pitfall 1)
- Curve: X25519 via `crypto/ecdh`

**Implementation steps:**

1. `GenerateX25519KeyPair() (*ecdh.PrivateKey, *ecdh.PublicKey, error)` — uses `ecdh.X25519().GenerateKey(rand.Reader)`

2. `Encrypt(plaintext []byte, senderPriv *ecdh.PrivateKey, senderPub *ecdh.PublicKey, recipientPub *ecdh.PublicKey, senderKID, recipientKID string) ([]byte, error)`:
   a. Generate ephemeral X25519 keypair
   b. Ze = ECDH(ephemeral_priv, recipient_pub) — ephemeral-static
   c. Zs = ECDH(sender_static_priv, recipient_pub) — static-static
   d. Generate random 64-byte CEK for A256CBC-HS512 (32 MAC key + 32 encryption key)
   e. Generate random 16-byte IV
   f. Encrypt plaintext with A256CBC-HS512: AES-256-CBC encrypt with enc_key, then HMAC-SHA512 truncated to 256 bits over (AAD || IV || ciphertext || AL)
   g. **Tag-in-KDF (CRITICAL per research Pitfall 2):** Z = Ze || Zs || tag (include ciphertext auth tag)
   h. Compute apu = base64url(senderKID), apv = base64url(sha256(recipientKID))
   i. Derive wrapping key = ConcatKDF(Z, "ECDH-1PU+A256KW", apu, apv, 256) using SHA-512
   j. Wrap CEK with AES-256-Key-Wrap (RFC 3394) using wrapping key
   k. Build JWE JSON: protected header (base64url), recipients array, iv, ciphertext, tag

3. `Decrypt(jweBytes []byte, recipientPriv *ecdh.PrivateKey, senderPub *ecdh.PublicKey) ([]byte, error)`:
   a. Parse JWE JSON, decode protected header
   b. Extract ephemeral public key (epk) from header
   c. Ze = ECDH(recipient_priv, ephemeral_pub), Zs = ECDH(recipient_priv, sender_static_pub) — note: recipient's perspective
   d. Z = Ze || Zs || tag (from JWE tag field)
   e. Derive wrapping key via same ConcatKDF
   f. Unwrap CEK via AES-256-Key-Unwrap
   g. Decrypt and verify with A256CBC-HS512

4. Internal helpers:
   - `concatKDF(z []byte, alg string, apu, apv []byte, keyLenBits int) []byte` — NIST SP 800-56C with SHA-512
   - `aesKeyWrap(kek, plaintext []byte) ([]byte, error)` — RFC 3394
   - `aesKeyUnwrap(kek, ciphertext []byte) ([]byte, error)` — RFC 3394
   - `encryptA256CBCHS512(key, iv, plaintext, aad []byte) (ciphertext, tag []byte, error)` — A256CBC-HS512
   - `decryptA256CBCHS512(key, iv, ciphertext, aad, tag []byte) ([]byte, error)`

**Do NOT use go-jose for X25519 ECDH** (it does not support X25519 — per research Pitfall 6). Use `crypto/ecdh` for all X25519 operations. Use `crypto/aes` for AES-CBC and AES-KW. Use `crypto/hmac` + `crypto/sha512` for HMAC and ConcatKDF.

**JWE JSON structure** (per DIDComm v2.1 spec):
```json
{
  "protected": "<base64url>",
  "recipients": [{"header": {"kid": "<recipientKID>"}, "encrypted_key": "<base64url>"}],
  "iv": "<base64url>",
  "ciphertext": "<base64url>",
  "tag": "<base64url>"
}
```

Protected header fields: alg, enc, epk (kty, crv, x), apu, apv, skid.
