    - CleanupExpiredMessages() deletes messages past expires_at

    - ValidateRecipientDomain(kid, platformDomain) checks that the DID in the kid belongs to this server

  <action>
**Create platform/internal/didcomm/mediator.go:**

1. `ExtractRecipientKID(jweBytes []byte) (string, error)` — parse JWE JSON (just the `recipients` array), return `recipients[0].header.kid`. Error if no recipients or missing kid.

2. `ExtractSenderKID(jweBytes []byte) (string, error)` — parse JWE JSON, decode protected header (base64url), return `skid` field. Error if missing.

3. `ValidateRecipientDomain(recipientKID, platformDomain string) bool` — the kid is a DID fragment like `did:web:atap.app:agent:01xxx#key-x25519-1`. Extract the domain portion (third segment of `did:web:`) and compare with platformDomain. Return true if match. This prevents the server from accepting messages for foreign DIDs (Pitfall 5 from research).

**Create platform/internal/store/messages.go:**

Implement `MessageStore` interface on `*Store`:

1. `QueueMessage` — INSERT into didcomm_messages (id, recipient_did, sender_did, message_type, payload, state, created_at, expires_at)
2. `GetPendingMessages` — SELECT ... WHERE recipient_did = $1 AND state = 'pending' ORDER BY created_at ASC LIMIT $2
3. `MarkDelivered` — UPDATE didcomm_messages SET state = 'delivered', delivered_at = NOW() WHERE id = ANY($1) AND state = 'pending'
4. `CleanupExpiredMessages` — DELETE FROM didcomm_messages WHERE expires_at < NOW() AND state = 'pending'

  <name>Task 2: POST /v1/didcomm and GET /v1/didcomm/inbox handlers</name>
  <files>platform/internal/api/didcomm_handler.go, platform/internal/api/didcomm_handler_test.go, platform/internal/api/api.go, platform/cmd/server/main.go</files>
  <behavior>
    - POST /v1/didcomm with Content-Type application/didcomm-encrypted+json and valid JWE returns 202 Accepted
    - POST /v1/didcomm with wrong Content-Type returns 415 Unsupported Media Type
    - POST /v1/didcomm with JWE addressed to unknown/foreign DID returns 400
    - POST /v1/didcomm with empty body returns 400
    - GET /v1/didcomm/inbox (DPoP auth) returns paginated list of pending messages for authenticated entity
    - GET /v1/didcomm/inbox marks returned messages as delivered
    - POST /v1/didcomm publishes to Redis inbox:{entity_id} for live notification
  </behavior>
  <action>
**Create platform/internal/api/didcomm_handler.go:**

1. `HandleDIDComm(c *fiber.Ctx) error` — POST /v1/didcomm:
   a. Validate Content-Type is `application/didcomm-encrypted+json` — return 415 if not
   b. Read raw body (JWE bytes)
   c. Call `didcomm.ExtractRecipientKID(body)` to find recipient
   d. Call `didcomm.ValidateRecipientDomain(recipientKID, h.config.PlatformDomain)` — return `problem(400, "invalid-recipient", ...)` if foreign DID
   e. Extract DID from kid (strip fragment `#key-x25519-1`), look up entity via `h.entityStore.GetEntityByDID(ctx, recipientDID)` — return 400 if not found
   f. Optionally extract sender KID for logging (non-fatal if missing)
   g. Generate message ID: `"msg_" + ULID`
   h. Call `h.messageStore.QueueMessage(ctx, &models.DIDCommMessage{...})`
   i. PUBLISH to Redis `inbox:{entityID}` with message ID (best-effort, skip if Redis unavailable)
   j. Return 202 Accepted with `{"id": "msg_...", "status": "queued"}`

   **Authentication:** This endpoint does NOT require OAuth/DPoP. DIDComm messages are self-authenticating via ECDH-1PU. Rate-limit by IP (defer to Phase 4; note in comment).

2. `HandleInbox(c *fiber.Ctx) error` — GET /v1/didcomm/inbox:
   a. This route IS behind DPoP auth middleware (requires `atap:inbox` scope)
   b. Get authenticated entity DID from context (set by DPoPAuthMiddleware)
   c. Parse `?limit=N` query param (default 50, max 100)
   d. Call `h.messageStore.GetPendingMessages(ctx, entityDID, limit)`
   e. For each message, return `{id, sender_did, message_type, payload (base64), created_at}`
   f. Mark all returned message IDs as delivered via `h.messageStore.MarkDelivered(ctx, ids)`
   g. Return 200 with `{"messages": [...], "count": N}`
