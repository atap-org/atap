<objective>
Build the OAuth 2.1 authorization server with DPoP token binding, replacing the old Ed25519 signed request authentication.

Purpose: This is the authentication layer. Agents authenticate via Client Credentials + DPoP, humans via Authorization Code + PKCE + DPoP. Every API token is sender-constrained via DPoP proof JWTs, preventing token theft.

Output: Working OAuth token endpoint, authorization code flow, DPoP middleware protecting all authenticated routes.
</objective>

    10. Return: `{"access_token": "<jwt>", "token_type": "DPoP", "expires_in": 3600, "scope": "<granted scopes>"}`

    Also issue a refresh token (AUTH-06): same flow but token_type=refresh, expires_at = now + 90 days. Return in response as `refresh_token`.

    **Create `platform/internal/api/auth.go`:**

    `DPoPAuthMiddleware() fiber.Handler` (AUTH-01, AUTH-04):
    Validation sequence per RESEARCH.md Pattern 3:
    1. Extract `DPoP` header. If missing, return 401 with `invalid_dpop_proof` error.
    2. Parse DPoP proof JWT using `dpop.Parse()` from go-dpop library. Validate `htm` matches `c.Method()`, `htu` matches request URL. Use 60-second time window for `iat`.
    3. If parse fails, return 401 with `invalid_dpop_proof`.
    4. Check `jti` nonce not seen before: Redis GET `dpop:nonce:{jti}`. If exists, return 401 `dpop_proof_replay`. If not, SET with EX 300 (5 min TTL per RESEARCH.md Pitfall 3).
    5. Extract `Authorization` header. Must start with `DPoP ` (NOT `Bearer` -- see Anti-Pattern). If `Bearer`, return 401 with `invalid_token_type` and detail "Use DPoP token type, not Bearer".
    6. Extract token value after `DPoP ` prefix.
    7. Parse JWT access token using go-jose/v4. Verify signature against platform's public key. Extract claims (sub, cnf.jkt, scope, exp).
    8. Verify token not expired (exp > now).
    9. Look up token by jti in oauth_tokens store. Verify not revoked.
    10. Verify `cnf.jkt` from token matches JWK thumbprint from DPoP proof. If mismatch, return 401 `dpop_binding_mismatch`.
    11. Look up entity by DID (from sub claim). Set in `c.Locals("entity")`.
    12. Set scopes in `c.Locals("scopes")` as `[]string`.
    13. Call `c.Next()`.

    `RequireScope(scope string) fiber.Handler` (AUTH-05):
    1. Get scopes from `c.Locals("scopes")`.
    2. Check if required scope is in the list.
    3. If not, return 403 with `insufficient_scope` error.
    4. Call `c.Next()`.
