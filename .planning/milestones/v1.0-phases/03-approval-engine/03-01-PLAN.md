<objective>
Strip server-side approval storage, add revocation infrastructure, and update OAuth scopes.

Purpose: The spec v1.0-rc1 changed the ATAP server from an approval storage engine to a pure DIDComm transport. The server must not store approvals; it stores only entity records, credentials, and revocation lists. This plan deletes all 6 approval CRUD endpoints, the approval store, and the approvals table, replacing them with a revocation list API (POST + GET /v1/revocations) and the underlying revocation table.

Output: Working revocation API with database backing, no approval storage, correct OAuth scopes.
</objective>
