## Critical Integration Issues

### 1. BLOCKING: Signal CreatedAt JSON Field Mismatch

**Files:** `platform/internal/models/models.go:207`, `mobile/lib/core/models/signal.dart:33`

Go `Signal.CreatedAt` is tagged `json:"-"` (excluded from JSON). Flutter `Signal.fromJson` reads `json['created_at']` which is always null, causing `TypeError` on every signal display.

**Impact:** All signal display is broken — inbox view, SSE events, signal detail screen.
**Fix:** Change Go tag to `json:"created_at"` or change Flutter to read `json['ts']`.
**Affects:** MOB-02, SSE-01, SIG-04

### 2. FUNCTIONAL BUG: Inbox Pagination Parameter Mismatch

**Files:** `mobile/lib/providers/inbox_provider.dart:107`, `platform/internal/api/api.go:467`

Flutter sends `?cursor=X`, Go reads `?after=X`. Infinite scroll silently re-fetches page 1 forever.

**Fix:** Change Flutter `loadMore()` from `cursor=` to `after=`.
**Affects:** SIG-04, MOB-02

### 3. BUG: Webhook Retry Empty Payload

**File:** `platform/internal/api/webhook.go:187-196`

`pollRetries` re-enqueues without signal payload. Retry delivers empty body.

**Fix:** Give `WebhookWorker` access to `SignalStore` to fetch payload, or store payload in `delivery_attempts`.
**Affects:** WHK-03

### 4. MINOR BUG: RegisterHuman Race Condition

**File:** `platform/internal/api/human.go:70-73`

Concurrent claim redemptions return 500 instead of 409 — missing `errors.Is(err, store.ErrClaimNotAvailable)` check.

**Affects:** MOB-04 under concurrent load
