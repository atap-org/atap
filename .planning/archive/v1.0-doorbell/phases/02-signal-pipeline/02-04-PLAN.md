<!-- All API endpoints available after Plans 01-03 -->

Endpoints (all under /v1):
- POST /register — register agent, returns {uri, id, public_key, private_key, key_id}
- GET /entities/:entityId — public entity lookup
- GET /me — authenticated entity info
- POST /inbox/:entityId — send signal (auth, signed signal body)
- GET /inbox/:entityId — get inbox (auth, cursor pagination: ?after=&limit=)
- GET /inbox/:entityId/stream — SSE stream (auth, Last-Event-ID for replay)
- POST /entities/:entityId/webhook — register webhook URL (auth)
- POST /entities/:entityId/channels — create channel (auth)
- GET /entities/:entityId/channels — list channels (auth)
- DELETE /entities/:entityId/channels/:channelId — revoke channel (auth)
- POST /channels/:channelId/signals — inbound webhook (channel auth)

Auth: Ed25519 signed requests via crypto.SignRequest(privKey, keyID, method, path, ts)
Errors: RFC 7807 ProblemDetail

<task type="auto">
  <name>Task 2: Integration tests for full agent lifecycle</name>
  <files>platform/test/integration_test.go</files>
  <action>
Add the following integration tests to the file created in Task 1. All tests use the setupTestInfra helper.

**TestFullLifecycle** (TST-01 primary test):
1. Register agent A and agent B
2. A sends signal to B's inbox (sendSignal helper)
3. B polls inbox (GET /v1/inbox/{B-id}) — assert signal present with correct data
4. B polls with cursor — assert hasMore=false (only 1 signal)
5. Verify signal ID starts with "sig_"
6. Verify signal has correct route (origin=A URI, target=B URI)

**TestSSEStream:**
1. Register agent A and agent B
2. B opens SSE stream in a goroutine (GET /v1/inbox/{B-id}/stream with auth)
3. Wait briefly for connection to establish
4. A sends signal to B
5. Read from SSE stream — assert signal event received with correct ID
6. Close stream

**TestSSEReplay:**
1. Register A and B
2. A sends signal-1 and signal-2 to B
3. B opens SSE stream with Last-Event-ID: signal-1
4. Assert signal-2 is replayed immediately
5. A sends signal-3 — assert it arrives in real-time
6. Close stream

**TestSignalPersistence:**
1. Register A and B
2. A sends signal to B
3. Query store directly to verify signal row exists in PostgreSQL
4. Verify all fields match: route, signal body, trust block, context

**TestIdempotency:**
1. Register A and B
2. A sends signal with idempotency_key="test-idem-1"
3. Assert 202
4. A sends same signal again with same idempotency_key
5. Assert 409 Conflict
6. Verify only 1 signal in B's inbox

**TestWebhookDelivery** (requires an HTTP test server to receive webhook):
1. Register agent A
2. Start httptest.Server that records received requests
3. A registers webhook URL pointing to test server
4. Register agent B
5. B sends signal to A
6. Wait briefly, assert test server received POST with:
   - Signal payload in body
   - X-ATAP-Signature header present
   - Verify signature against platform public key

**TestChannelInbound:**
1. Register entity A
2. A creates an open channel
3. POST raw JSON to channel webhook URL with Basic Auth credentials
4. A polls inbox — assert signal present with source="external", trust_level=0
5. Verify the original payload is wrapped in signal data field

**TestChannelTrusted:**
1. Register entity A and entity B (trustee)
2. A creates trusted channel with trustee_id=B
3. B POSTs to channel webhook URL with B's Ed25519 signature
4. A polls inbox — assert signal present with correct trust info

**TestChannelRevoke:**
1. Register entity A, create channel
2. A revokes channel
3. POST to channel URL — assert 404 or 410

**TestExpiredSignalExcluded:**
1. Register A and B
2. A sends signal with TTL=1 (1 second)
3. Wait 2 seconds
4. B polls inbox — assert signal NOT returned (expired)

Run with: `cd platform && go test -tags integration ./test/... -count=1 -v -timeout 120s`

Note: These tests require Docker running. The -timeout 120s flag gives testcontainers time to pull images on first run.
