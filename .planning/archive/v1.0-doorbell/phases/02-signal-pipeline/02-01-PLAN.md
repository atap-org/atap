**002_signals.up.sql:**
```sql
CREATE TABLE signals (
    id                TEXT PRIMARY KEY,           -- sig_ + ULID
    version           TEXT NOT NULL DEFAULT '1',
    ts                TIMESTAMPTZ NOT NULL,

    -- Route
    origin            TEXT NOT NULL,              -- sender entity URI
    target            TEXT NOT NULL,              -- target entity URI
    target_entity_id  TEXT NOT NULL REFERENCES entities(id),
    reply_to          TEXT,
    channel_id        TEXT,                       -- channel that received it (if via channel)
    thread_id         TEXT,
    ref_id            TEXT,

    -- Trust
    trust_level       INTEGER NOT NULL DEFAULT 0,
    signer            TEXT NOT NULL,              -- signer entity URI
    signer_key_id     TEXT NOT NULL,
    signature         TEXT NOT NULL,              -- base64 Ed25519 signature

    -- Signal body
    signal_type       TEXT NOT NULL,
    encrypted         BOOLEAN NOT NULL DEFAULT false,
    data              JSONB,                      -- signal payload (max 64KB enforced in app)

    -- Context
    source_type       TEXT NOT NULL DEFAULT 'agent' CHECK (source_type IN ('agent', 'external', 'system')),
    idempotency_key   TEXT,
    tags              JSONB DEFAULT '[]',
    ttl               INTEGER,                    -- seconds, NULL = default 7 days
    priority          TEXT NOT NULL DEFAULT 'normal' CHECK (priority IN ('normal', 'high', 'urgent')),

    -- Server-side
    delivery_status   TEXT NOT NULL DEFAULT 'pending' CHECK (delivery_status IN ('pending', 'delivered', 'failed')),
    expires_at        TIMESTAMPTZ,               -- computed: ts + ttl (or ts + 7 days)
    created_at        TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

-- Inbox query: by target, ordered by ID (ULID = time-ordered), excluding expired
CREATE INDEX idx_signals_inbox ON signals(target_entity_id, id) WHERE delivery_status != 'failed';

-- Cursor pagination and SSE replay: signals after a given ID for a target
CREATE INDEX idx_signals_target_after ON signals(target_entity_id, id ASC);

-- Idempotency dedup: unique within 24h window (use app-level check since partial index on NOW() is not immutable)
CREATE UNIQUE INDEX idx_signals_idempotency ON signals(idempotency_key) WHERE idempotency_key IS NOT NULL;

-- TTL cleanup: find expired signals efficiently
CREATE INDEX idx_signals_expires ON signals(expires_at) WHERE expires_at IS NOT NULL;

-- Thread lookup
CREATE INDEX idx_signals_thread ON signals(thread_id) WHERE thread_id IS NOT NULL;
```

**002_signals.down.sql:** `DROP TABLE IF EXISTS signals;`

**003_channels.up.sql:**

```sql
CREATE TABLE channels (
    id              TEXT PRIMARY KEY,           -- chn_ + 32 hex
    entity_id       TEXT NOT NULL REFERENCES entities(id),
    label           TEXT,
    tags            JSONB DEFAULT '[]',
    type            TEXT NOT NULL CHECK (type IN ('trusted', 'open')),
    trustee_id      TEXT,                      -- for trusted channels: entity ID of trustee

    signal_count    BIGINT NOT NULL DEFAULT 0,
    created_at      TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    expires_at      TIMESTAMPTZ,
    revoked_at      TIMESTAMPTZ
);

CREATE INDEX idx_channels_entity ON channels(entity_id) WHERE active = true;
CREATE INDEX idx_channels_trustee ON channels(trustee_id) WHERE trustee_id IS NOT NULL AND active = true;

CREATE TABLE webhook_configs (
    entity_id   TEXT PRIMARY KEY REFERENCES entities(id),
    url         TEXT NOT NULL,
    created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

Extend store.go with the following methods (append after existing entity methods, following the same pgx/v5 pool pattern):

**Signal methods:**

`SaveSignal(ctx, s *models.Signal) error` — INSERT into signals table. Compute expires_at from s.TS + TTL (use DefaultSignalTTL if TTL is 0). For idempotency: use INSERT ... ON CONFLICT (idempotency_key) DO NOTHING, then check rows affected — if 0, return a sentinel error `ErrDuplicateSignal` (define as `var ErrDuplicateSignal = fmt.Errorf("duplicate signal")`) so the handler can return 409.

`GetSignal(ctx, id string) (*models.Signal, error)` — SELECT by primary key, return nil if not found (same pgx.ErrNoRows pattern as GetEntity).

`GetInbox(ctx, entityID, after string, limit int) ([]*models.Signal, bool, error)` — SELECT signals for target_entity_id, WHERE expires_at IS NULL OR expires_at > NOW(), ordered by id ASC. If `after` is non-empty, add AND id > after. Fetch limit+1 rows to determine hasMore. Return signals[:limit], hasMore, nil.

`GetSignalsAfter(ctx, entityID, afterID string) ([]*models.Signal, error)` — Same as GetInbox but no limit (for SSE replay). Use a reasonable cap of 1000 to prevent memory issues.

**Channel methods:**

`CreateChannel(ctx, ch *models.Channel) error` — INSERT into channels. For open channels, basic_auth_hash should already be set by the handler.

`GetChannel(ctx, id string) (*models.Channel, error)` — SELECT by primary key.

`GetChannelByID(ctx, id string) (*models.Channel, error)` — Alias or same as GetChannel, returns nil if not found or not active.

`ListChannels(ctx, entityID string) ([]*models.Channel, error)` — SELECT WHERE entity_id = $1 AND active = true, ORDER BY created_at DESC.

`RevokeChannel(ctx, id string) error` — UPDATE channels SET active = false, revoked_at = NOW() WHERE id = $1.

`IncrementChannelSignalCount(ctx, channelID string) error` — UPDATE channels SET signal_count = signal_count + 1 WHERE id = $1.

**Webhook config methods:**

`GetWebhookConfig(ctx, entityID string) (*models.WebhookConfig, error)` — SELECT by entity_id, return nil if not found.

`SetWebhookConfig(ctx, entityID, url string) error` — INSERT ... ON CONFLICT (entity_id) DO UPDATE SET url = $2, updated_at = NOW().

`DeleteWebhookConfig(ctx, entityID string) error` — DELETE FROM webhook_configs WHERE entity_id = $1.

**Delivery attempt methods:**

`SaveDeliveryAttempt(ctx, a *models.DeliveryAttempt) error` — INSERT into delivery_attempts.

`GetPendingRetries(ctx, now time.Time) ([]*models.DeliveryAttempt, error)` — SELECT WHERE next_retry_at IS NOT NULL AND next_retry_at <= $1, LIMIT 100.

`UpdateSignalDeliveryStatus(ctx, signalID, status string) error` — UPDATE signals SET delivery_status = $2 WHERE id = $1.

`CleanupDeliveryAttempts(ctx, olderThan time.Time) (int64, error)` — DELETE FROM delivery_attempts WHERE created_at < $1, return rows affected.

**Scanning helper:** Create a private `scanSignal(row)` function that scans a pgx.Row or pgx.Rows into a *models.Signal, similar to how entity fields are scanned inline in existing methods. This avoids repeating the long column list.

All methods must wrap errors with context: `fmt.Errorf("action: %w", err)`.
